3. Brand-Store Fit

"Strong store + Strong brand" = 45.3% reorder (9,749 SKU) – nejvyšší riziko. Prodejna je silná v tomto brandu → odvezené se prodá a musí se doobjednat.
"Weak store + Weak brand" = 29.3% reorder (7,596 SKU) – nejnižší riziko. Tyto redistribuce fungují nejlépe.

Rozdíl: 16 procentních bodů! Brand-store fit je silný prediktor.

4. Navrhovaný rozhodovací strom pro Source MinLayer

Pravidla v pseudokódu:

FUNCTION CalculateSourceMinLayer(sku):
    base = current_min_layer_3_value  // z heuristiky

    // 1. Delisting override
    IF sku.SkuClass IN (D, L, R):
        RETURN 0  // bezpečné odvézt vše

    // 2. Frekvence prodejů (hlavní faktor)
    IF sku.Sales6M == 0:
        IF store.Decile >= 8:   base = MAX(base, 2)
        ELIF store.Decile >= 4: base = MAX(base, 2)
        ELSE:                    base = MAX(base, 1)
    ELIF sku.Sales6M <= 2:
        base = MAX(base, 2)
        IF store.Decile >= 8:   base = MAX(base, 3)
    ELSE:  // Sales6M >= 3
        base = MAX(base, 3)

    // 3. Modifikátory
    IF product.Trend == 'Growing':     base += 1
    IF brand_store_fit == 'Strong+Strong': base += 1
    IF sku.StockoutDays_6M > 90:       base += 1
    IF sku.RedistRatio > 0.75:         base = MAX(base - 1, 0)  // odvážíme skoro vše
    IF product.Trend == 'Declining':   base = MAX(base - 1, 0)

    RETURN MIN(base, 5)  // cap na 5

Odhad dopadu navrhovaných pravidel:

Při navýšení MinLayer by se:

  • {skus_changed:,} SKU ({skus_changed/total_skus*100:.1f}%) dostalo vyšší MinLayer → méně by se redistribuovalo (nebo vůbec)
  • {skus_unchanged:,} SKU ({skus_unchanged/total_skus*100:.1f}%) by zůstalo stejných
  • Odhad snížení reorderu: pokud by zvýšení MinLayer o 1 zabránilo redistribuci u SKU s 1-2 prodejemi → ušetříme ~4,000-6,000 zbytečných reorderů

Poznámka: Přesný backtest vyžaduje re-run kalkulačního engine s novými pravidly. Výše je konzervativní odhad.
